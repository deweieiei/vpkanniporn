▌ npm packages 5 ตัว — declare ไว้ใน package.json แต่ไม่มีจุดไหน require()
  เลยสักครั้งในโค้ดทั้งโปรเจกต์:

  1. axios        — ไม่ได้ใช้ (โปรเจกต์เรียก API ภายในตัวเองผ่าน fetch() ทั้งหมด)
  2. form-data     — ไม่ได้ใช้ (ไม่มี multipart request ออกไปหาที่อื่น)
  3. mssql         — ไม่ได้ใช้ (driver MS SQL Server — โปรเจกต์นี้ใช้ MySQL เท่านั้น)
  4. pg            — ไม่ได้ใช้ (driver PostgreSQL — ไม่เกี่ยวกับโปรเจกต์นี้เลย)
  5. tedious       — ไม่ได้ใช้ (dependency ของ mssql — พลอยติดมาด้วย)

  ★ 3 ตัวหลัง (mssql, pg, tedious) น่าจะติดมาจากตอน generate scaffold
    ครั้งแรก ไม่เคยถูกใช้งานจริงเลยสักบรรทัด

  ผลลัพธ์หลังลบ: `npm uninstall` ลบไปทั้งหมด 102 packages (รวม transitive
  dependencies ที่ตามมาด้วย) node_modules เบาลงเยอะมาก, npm install
  รอบต่อไปจะเร็วขึ้น ไม่กระทบการทำงานของเว็บเลย (ทดสอบรัน server แล้วปกติ)

  ไฟล์ที่เปลี่ยน: package.json, package-lock.json

▌ multer 2.1.1 มีช่องโหว่ระดับ High 2 จุด (Denial of Service):
  - ส่ง field name ซ้อนลึกมากๆ ทำให้ server ค้าง/ล่ม
  - ไฟล์อัปโหลดที่ถูกยกเลิกกลางทาง ไม่ถูกเคลียร์ให้หมด (ทรัพยากรรั่ว)
  อัปเดตเป็น multer 2.2.0 (patch version เดียวกัน ไม่กระทบโค้ดเดิม)
  `npm audit` หลังแก้ = 0 vulnerabilities
