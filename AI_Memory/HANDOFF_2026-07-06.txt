■ โปรเจกต์นี้คืออะไร
  เว็บสำนักงานตัวแทนประกัน FWD ของคุณกวิสรา จงเจริญ ("VP Kanniporn")
  Deploy จริงแล้วที่ https://vpkanniporn.com (หลัง Cloudflare + HostAtom/Plesk)
  Repo: https://github.com/deweieiei/vpkanniporn (branch main, public)

■ Stack
  Node.js + Express 4 + MySQL + express-session (เก็บ session ใน MySQL) + multer
  Frontend เป็น HTML/JS ล้วน (บางหน้าใช้ Tailwind CDN, บางหน้า CSS เอง)
  รันเครื่อง dev: npm run dev (port 3000) — .claude/launch.json ชื่อ vpkanniporn-dev

■■■ สถาปัตยกรรมปัจจุบัน (หลังรื้อใหญ่รอบ 14-15) ■■■

  แนวคิด: เว็บ = ไดเรกทอรีตัวแทน ให้คนค้นหาตัวแทน + ตัวแทนมีโปรไฟล์/หน้าของตัวเอง
  SupperAdmin เป็นคนสร้างบัญชีตัวแทน (ไม่มีสมัครเอง)

  หน้าเว็บ (routes ใน routes/pages.js):
    /                = หน้า login (login.html)   *** ดู "ค้างตัดสินใจ" ข้อ 1 ***
    /login           = หน้า login (เหมือน /)
    /home            = หน้าค้นหาตัวแทน (home.html) — เดิมชื่อ index.html
    /agent/:id       = โปรไฟล์ตัวแทน (dashboard.html โหมด public อ่านอย่างเดียว)
    /profile /dashboard = โปรไฟล์ตัวเอง (ต้อง login, แก้ไขได้)
    /page/:id        = หน้าเนื้อหาของ "ปุ่ม" ที่ตัวแทนสร้าง (page.html)
                       คนทั่วไปเห็นเนื้อหา / เจ้าของที่ login เห็น toolbar แก้ไข
    /support-admin        = login ของ SupperAdmin (supperadmin-login.html)
    /support-admin/users  = หน้าจัดการตัวแทน (supperadmin-users.html)
    /plans /plans/:id     = แบบประกัน Phase 3 (อาจไม่ได้ใช้แล้วในทิศทางใหม่)
    /search               = หน้าค้นหาเก่า (ซ้ำกับ /home — พิจารณาลบ)

  บทบาท (users.role): admin / agent / user / support_admin
    - "SupperAdmin" ใช้ role = 'admin' (login ด้วยชื่อผู้ใช้ ไม่ใช่อีเมล)
    - ตัวแทน = role 'agent' (login ด้วยอีเมล+รหัส ที่ /login)

  Session: "ไม่จำ login" — เป็น session cookie (ปิดเบราว์เซอร์ต้องล็อกอินใหม่)
    ตั้งใน index.js: rolling=false + ไม่ตั้ง cookie.maxAge

  API:
    /api/auth/login (อีเมล), /api/auth/supperadmin-login (username), logout, me, PUT profile
    /api/users (list), /api/users/:id
    /api/admin/users  GET/POST/PUT/DELETE  (จัดการตัวแทน — เฉพาะ role admin)
    /api/pages        ปุ่ม/หน้าของตัวแทน:
        GET /api/pages?user_id=N (public list), GET /api/pages/mine (เจ้าของ),
        GET /api/pages/:id (public + is_owner), POST, PUT /:id, POST /:id/image, DELETE /:id
    /api/plans, /api/categories (Phase 3)

■■■ ฐานข้อมูล (ตาราง) ■■■
    users                 — ผู้ใช้ทั้งหมด (role enum มี support_admin แล้ว)
    insurance_categories / insurance_plans — Phase 3 แบบประกัน
    user_pages            — ปุ่ม/หน้าที่ตัวแทนสร้างเอง (ใหม่ รอบ 15)
    site_content          — *เลิกใช้แล้ว* (CMS ถูกยกเลิกรอบ 14) เหลือค้างใน DB เฉยๆ ลบได้
    sessions              — express-mysql-session สร้างอัตโนมัติ

  *** SQL ที่ต้องรันบน DB จริง (ลำดับ) — ไฟล์อยู่ใน db/ ***
    1. db/schema.sql                    (users + insurance tables) — ถ้า DB ใหม่
    2. db/seed_admin.sql                (บัญชี admin/1234)  ← หรือ migration_supperadmin.sql
    3. db/migration_insurance_plans.sql (ถ้ายังไม่ได้รัน)
    4. db/migration_user_pages.sql      (ตาราง user_pages)
  *** พี่ดิวรัน node script เองไม่ได้ — ให้ทำทุกอย่างผ่าน SQL (phpMyAdmin) เสมอ ***

■■■ บัญชีเข้าระบบ ■■■
    SupperAdmin: /support-admin  →  user: admin  /  รหัส: 1234  (bcrypt แล้ว, ควรเปลี่ยนหลัง deploy)
    ตัวแทน: SupperAdmin สร้างให้ที่ /support-admin/users แล้ว login ที่ /login
